# ATracker

Небольшой кроссплатформенный (macOS + Windows) CLI, который работает фоновым процессом.
Раз в интервал он показывает нативный диалог с вопросом «Чем ты занимался?» и периодом,
за который спрашивают (например `(с 13:05:20 по 14:05:20)`), сохраняет ответ в журнал и
умеет строить отчёт за день или неделю — куда ушло время.

## Установка

Требуется Node.js 20+.

```bash
npm install      # установить зависимости
npm run build    # скомпилировать TypeScript в dist/
npm link         # сделать команду `atracker` доступной глобально
```

Для разработки можно запускать без сборки:

```bash
npm run dev -- report          # = atracker report
npx tsx src/index.ts report    # то же самое
npm test                       # юнит-тесты (storage, report)
```

## Команды

| Команда | Описание |
| --- | --- |
| `atracker start` | Запустить фоновый процесс (откажется, если уже запущен). |
| `atracker stop` | Остановить фоновый процесс и удалить файл состояния. |
| `atracker log "текст" [--time HH:MM]` | Добавить запись вручную. `--time` ставит запись задним числом (сегодняшняя дата). |
| `atracker report [--date YYYY-MM-DD \| --week]` | Отчёт: за сегодня / за конкретную дату / за последние 7 дней. |
| `atracker install` | Зарегистрировать автозапуск при входе в систему (macOS LaunchAgent, Windows schtasks — без прав администратора). |

### Пример отчёта

```text
10:00 написал письма
11:00 код ревью
12:00 код ревью  ← скопировано (закрыл диалог)
13:00 нет ответа ← истёк таймаут
14:00 отсутствует ← компьютер спал
15:00 звонок с командой
```

## Четыре состояния

Это суть приложения. У каждой записи есть `timestamp` и `status`. На диск
записываются только **три** явных статуса; четвёртое состояние (пропуск) нигде не
хранится и выводится в отчёте только при его построении.

| Состояние | Хранится на диске? | Как выглядит в отчёте |
| --- | --- | --- |
| `answered` | да, с полем `activity` | сам текст активности |
| `dismissed` | да (без копии текста) | предыдущая активность `← скопировано (закрыл диалог)` |
| `timeout` | да | `нет ответа ← истёк таймаут` |
| пропуск (компьютер спал/выключен) | **нет** | `отсутствует ← компьютер спал` |

Хранилище (`storage.ts`) фиксирует только сырые факты. Вся интерпретация —
перенос предыдущей активности (`dismissed`) и заполнение пропусков
(`delta > interval × 1.5`) — происходит только на уровне отчёта (`report.ts`) и
никогда не записывается обратно.

## Файлы данных (`~/.atracker/`)

| Файл | Назначение |
| --- | --- |
| `config.json` | Единый источник настроек, читается всеми частями приложения. |
| `process.json` | Состояние демона: `{ pid, startedAt }`. Используется для защиты от двойного запуска. |
| `activities.jsonl` | Журнал, по одному JSON-объекту на строку (append-only). |

### Настройки (`config.json`)

| Поле | По умолчанию | Описание |
| --- | --- | --- |
| `intervalMinutes` | `60` | Интервал между диалогами. Тот же интервал используется для определения пропусков и группировки в отчёте. |
| `dialogTimeoutMinutes` | `10` | Сколько диалог ждёт ответа, прежде чем закрыться со статусом `timeout`. |
| `question` | `"Чем ты занимался?"` | Текст вопроса в диалоге. К нему автоматически добавляется период, например `(с 13:05:20 по 14:05:20)`. |

Файл создаётся автоматически со значениями по умолчанию при первом запуске любой команды.
